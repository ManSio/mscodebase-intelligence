# Архитектурный аудит: MCP vs IDE-Native

> Дата: 2026-07-14
> Проект: MSCodeBase Intelligence
> Цель: Сравнительный анализ MCP-инструментов и IDE-native (grep/read_file/terminal)

---

## 1. Методология

Два независимых агента выполнили идентичный аудит проекта:
- **Агент A** — используя MCP-инструменты (search_code, get_symbol_info, read_live_file, intel_*)
- **Агент B** — используя только grep, read_file, terminal

Оба собрали: системные ресурсы, тайминги операций, архитектурную структуру, риски.

## 2. ⏱ Сравнение производительности

| Операция | MCP (ms) | IDE-Native (ms) | Разница |
|----------|---------|-----------------|---------|
| Поиск/чтение файла | 0.8 (`read_live_file`) | 0.3 (`read_live_file`) | MCP медленнее ×2.7 |
| Поиск по смыслу | 4095 (`search_code`) | 138 (`grep`) | MCP медленнее ×30 |
| Инфо о символе | 1005 (`get_symbol_info`) | 21 (`grep`) | MCP медленнее ×48 |
| Git log | 24 (`git(action=log)`) | 49 (`git log`) | **MCP быстрее ×2** |
| Health report | 3780 (`get_health_report`) | 91 (проверка портов) | MCP медленнее ×42 |
| Структурный поиск | 6832 (`structural_search`) | 38 (Python os.walk) | MCP медленнее ×180 |
| Поиск bare except | — (нет инструмента) | 91 (`grep except:`) | MCP не умеет |
| Индекс состояния | 521 (`intel_get_runtime_status`) | ~50 (wc -l) | MCP медленнее ×10 |
| **Итого (8 операций)** | **~15 800 ms** | **~424 ms** | **MCP медленнее ×37** |

> **Важно:** MCP делает семантический поиск (эмбеддинги + вектора), grep — только regex.
> Сравнение НЕ полностью честное, но наглядно показывает overhead ML-пайплайна.

## 7. 🏁 Выводы

### MCP — это ML-слой над кодом
- Проигрывает grep в скорости ×30–180
- Жрёт ×50 больше RAM
- **Но даёт то, чего grep не может:** семантику, граф вызовов, hotspots, root cause

### IDE-Native — хирургическая точность
- Идеален для точных паттернов (bare except, hardcoded IP)
- Подсчёт метрик (строки, вхождения)
- Быстрое прототипирование (нет оверхэда ML)

### Практический вывод
**Для аудита эффективнее гибридный подход:**
- **MCP** → semantic search, граф вызовов, hotspots, root cause
- **grep** → точный подсчёт, bare except, hardcoded, размеры файлов

Их комбинация даёт **полную картину за минимальное время**.
