# 系统要求与架构参考

> 基于在 Windows 11 Insider Preview 构建版本 26220 上的实际基准测试（2026-07-10）
> CPU：AMD Ryzen 5 5600H（12 核心）| RAM：16 GB DDR4

---

## 1. 最低要求

| 组件 | 最低配置 | 推荐配置 |
|-----------|---------|-------------|
| **CPU** | 支持 AVX2 的 x86_64（AMD Ryzen 3xxx / Intel Haswell 4xxx+） | 任意支持 AVX2 的 x86_64 |
| **RAM** | 8 GB（系统）+ 1.2 GB（MCP + 模型） | 16 GB + 1.2 GB |
| **存储** | 2 GB 空闲（模型 + 索引） | 5 GB（适用于大型单体仓库） |
| **操作系统（Windows）** | Windows 10 22H2+ | Windows 11 稳定版（非 Insider Preview） |
| **操作系统（macOS）** | macOS 12+ Intel 或 Apple Silicon | macOS 14+ Apple Silicon（Metal） |
| **操作系统（Linux）** | Ubuntu 22.04+ x86_64 | Ubuntu 24.04+ |
| **Python** | 3.10+ | 3.12+ |
| **Zed** | v1.10.0+ | 最新版 |

## 2. RAM 占用明细（实测数据）

### 当前架构（通过 llama.cpp GGUF 运行 Qwen3 + BGE-M3）

| 进程 | RAM（无 --mlock） | RAM（有 --mlock） | 说明 |
|---------|-----------------|-------------------|-------|
| **MCP（python.exe）** | 213 – 252 MB | 213 – 252 MB | 稳定，无内存泄漏 |
| **llama-server Qwen3**（端口 8080） | **346 MB** | 772 MB | 嵌入模型 |
| **llama-server BGE-M3**（端口 8081） | **450 MB** | 539 MB | 重排序器（reranker）模型 |
| **总计** | **~1.0 – 1.1 GB** | ~1.5 – 1.6 GB | |

### 历史对比

| 日期 | MCP RAM | 模型 RAM | 总计 | 配置 |
|------|---------|-----------|-------|--------|
| 2026-07-05 | 185 MB | 0 MB（外部 API） | **185 MB** | LM Studio（远程） |
| 2026-07-08 | 172 MB | 0 MB（外部 API） | **172 MB** | LM Studio |
| 2026-07-09（早期） | 151 MB | 0 MB（外部 API） | **151 MB** | LM Studio 不可用 |
| 2026-07-09（后期） | 1,931 MB | 757 MB | **2,688 MB** | ONNX 进程内 🔴 |
| **2026-07-10** | **252 MB** | **796 MB** | **~1,048 MB** | **Qwen3 + BGE-M3 GGUF** 🟢 |

**关键发现：** MCP 进程中的 ONNX Runtime 导致 RAM 超过 2 GB。迁移到 llama.cpp GGUF 后，MCP 内存减少了 88%，总内存减少了 60%。
