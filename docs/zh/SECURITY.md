如果您在 MSCodeBase Intelligence 中发现安全漏洞，请负责任地报告。

1. **GitHub 安全公告**：[创建建议](https://github.com/ManSio/mscodebase-intelligence/security/advisories/new)

- 漏洞描述（类型、组件、利用条件）

- 概念验证（如适用）
- 修复建议（可选）

## 安全架构

### 混合架构（LSP + MCP）

从版本 2.0.0 开始，MSCodeBase Intelligence 使用结合了语言服务器协议（LSP）和模型上下文协议（MCP）的混合架构：

- **LSP 服务器**：负责通过 Tree-sitter 解析代码并向编辑器提供符号
- **MCP 服务器**：提供语义搜索、嵌入和与 AI 代理的交互
- **交互**：两个组件都在本地运行，通过 stdio/SSE 交换数据

### 什么会离开您的计算机

**默认情况下 — 什么都不会。** 所有组件完全在本地运行：

| 组件 | 数据 | 发送到何处 |
|-----------|--------|-------------------|
| LanceDB | 向量索引、代码块 | 本地（`.codebase_indices/`） |
| Tree-sitter | AST 解析 | 本地，进程内存中 |
| SafePathManager | 路径验证 | 本地，无网络 |
| MCP 工具 | 搜索/分析请求 | 本地，stdio/SSE |

**唯一例外：** 如果配置了 LM Studio 或 Ollama 集成用于生成嵌入 — 请求仅发送到本地端点（LM Studio 为 `localhost:1234`，Ollama 为 `localhost:11434`）。不使用云 API。

### 输入数据保护

- **所有 MCP 工具的验证**：每个工具检查输入参数的类型、长度和格式
- **路径遍历保护**：`SafePathManager` 阻止超出项目目录的访问（`../`、项目外的绝对路径）
- **SQL 注入**：LanceDB 使用参数化查询，用户输入不会插值到查询中
- **输入数据大小**：查询长度和索引文件大小的限制

### 数据保护

- **本地存储**：所有索引存储在项目内的 `.codebase_indices/` 中
- **路径哈希**：项目路径被哈希以实现索引隔离
- **过滤**：`.gitignore` 中的文件和二进制文件被排除在索引之外
- **加密**：数据在磁盘上不加密（假设文件系统受操作系统保护）

### 访问控制

- **文件系统**：访问限制在项目目录内
- **权限**：不需要提升的权限（不需要 `sudo`/管理员）
- **网络访问**：不需要即可工作（除了本地 LM Studio/Ollama）

## 混合架构安全（2.0.0+）

### 风险和缓解措施

| 风险 | 描述 | 缓解措施 |
|------|----------|-----------|
| LSP 服务器可通过网络访问 | 如果在 SSE 模式下运行，可能从外部访问 | 默认：stdio；SSE 仅在 `localhost` 上 |
| MCP 工具接受任意输入 | 恶意代理可能传递无效数据 | 每个工具级别的验证 |
| 通过嵌入泄露代码 | 代码发送到 LM Studio 进行索引 | 仅本地 LM Studio；不支持云服务 |
| 索引时的竞态条件 | 并发写入 LanceDB | 块版本控制、原子更新 |

### 部署建议

1. 如果不需要远程访问，**使用 stdio 模式**用于 LSP 和 MCP
2. **LM Studio/Ollama**：确保服务器仅监听 `127.0.0.1`，而不是 `0.0.0.0`
3. **防火墙**：阻止来自 MSCodeBase 进程的出站连接（如果不使用云服务）
4. **更新**：通过 Dependabot 关注依赖项更新

## 依赖项

- **GitHub Dependabot**：自动依赖项扫描

### 关键依赖项

| 依赖项 | 用途 | 安全说明 |
|-------------|------------|---------------------------|
| LanceDB | 向量数据库 | 本地存储，无网络访问 |
| Tree-sitter | 代码解析 | 沙箱化，不执行任意代码 |
| LM Studio SDK | 嵌入生成 | 仅本地端点 |
| Ollama SDK | 替代嵌入 | 仅本地端点 |
