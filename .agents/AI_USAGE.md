# MSCodeBase Intelligence — Инструкция для AI-ассистента

## Инструменты MCP-сервера

| Инструмент | Назначение | Когда использовать |
|------------|------------|-------------------|
| `index_project_dir(path)` | Индексация проекта | При первом открытии или после изменений |
| `search_code(query)` | Семантический поиск | Поиск функций, классов, методов по смыслу |
| `get_context(query)` | Сбор контекста | Сбор релевантных фрагментов кода |
| `get_symbol_info(query)` | Инфо о символе | Найти определение, вызовы, зависимости |
| `get_repo_map(project_root)` | Карта проекта | Понимание структуры |
| `get_index_status()` | Статус индекса | Проверка готовности |
| `scan_changes(project_root)` | Архитектурный дифф | Изменения вне Zed |
| `watcher_status()` | Статус системы | Проверка здоровья |

## Алгоритм работы

1. **Проверь статус**: `get_index_status()`
2. **Если пусто** — запусти индексацию: `index_project_dir(path=...)`
3. **Ищи и анализируй** через `search_code` и `get_context`
4. **Для рефакторинга** — используй `get_symbol_info` для поиска всех зависимостей
5. **После изменений** — `scan_changes` для отслеживания

## Важные ограничения

- Файлы > 1 MB пропускаются
- Игнорируются: `.git`, `node_modules`, `venv`, `__pycache__`, `.zed`, `.vscode`
- Требуется LM Studio (или Ollama/ONNX fallback) для эмбеддингов
- Поддерживаемые языки: Python, Rust, TypeScript, JavaScript, Go
- Пути нормализуй в POSIX lowercase: `path.as_posix().lower()`
