# MSCodebase Intelligence — Быстрый старт за 3 минуты

> Три шага, чтобы начать искать код через ИИ в Zed.

---

## 🪜 Шаг 1: Установка

```cmd
:: Откройте Git Bash в папке с расширением и запустите:
install.bat
```

Установщик сам:
- Создаст venv и установит зависимости
- Настроит `settings.json` Zed
- Пропишет путь к вашему проекту

**После установки:** `Ctrl+Shift+P` → `window: reload`

---

## 🪜 Шаг 2: Первая индексация

Откройте любой файл вашего проекта в Zed.

Нажмите `Ctrl+Shift+P` → **Agent Panel: Toggle**

В поле ввода Agent Panel напишите:
```
index_project_dir(path="D:\Project\MyProject")
```

*Замените путь на ваш проект.*

🔄 **Подождите 1-2 минуты** — идёт первичная индексация.

Проверить статус:
```
get_index_status()
```
Должны увидеть `📊 Статус базы данных` с числом чанков > 0.

---

## 🪜 Шаг 3: Первый поиск

Теперь можно искать код. Попробуйте:

```
search_code("класс или функция отвечающие за авторизацию", mode="quality")
```

Или просто спросите по-русски:

> "Найди все файлы, которые связаны с обработкой ошибок"

> "Покажи граф вызовов для функции send_message"

> "Какие файлы чаще всего меняются вместе с main.py?"

---

## ✅ Что дальше?

| Если хотите... | Попробуйте |
|----------------|------------|
| Узнать состояние сервера | `get_index_status()` |
| Посмотреть логи | `get_logs(project_root="D:\Project")` |
| Найти баги в коде | `get_hotspots(project_root="D:\Project")` |
| Понять зависимости | `graph_query(query_type="deps", target="src/main.py")` |
| Полную диагностику | `get_health_report(project_root="D:\Project")` |

---

## 🔧 Если что-то пошло не так

**Симптом:** Agent Panel пишет «MCP server not found»
→ **Решение:** `Ctrl+Shift+P` → `window: reload`

**Симптом:** `0 chunks` в статусе
→ **Решение:** Вызовите `index_project_dir(path)` и подождите

**Симптом:** Всё работает, но поиск пустой
→ **Решение:** Убедитесь что индексация завершена (`get_index_status` → chunks > 0)
