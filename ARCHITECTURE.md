* **Видеокарта (GPU):** AMD Radeon(TM) Graphics / Honor Virtual Display Device.
* **Дисковая подсистема:** Скоростной SSD-накопитель, выделенный тестовый логический диск `D:` объемом **341 GB** (файловая система NTFS).
* **Операционная система:** Майкрософт Windows 11 Домашняя Insider Preview (с жестким контролем нормализации путей, регистра дисков `D:\` vs `d:\` и системных слэшей `\` и `/`).

### 2. Спецификация Векторного Окружения и Моделей

Расширение реализует двухкомпонентную схему векторного поиска (Dense Retrieval):

* **Модель эмбеддингов:** `text-embedding-bge-m3`.
* **Протокол взаимодействия:** Локальный/удаленный API-сервер, эндпоинт векторизации: `/v1/embeddings`.
* **Стратегия превышения контекста:** Автоматическое разделение исходного кода на чанки с логированием факта обрезки длинных строк (`Truncated in logs`).
* **СУБД хранилища:** LanceDB v2 (на базе высокопроизводительного формата Apache Arrow).

### 3. Топология Распределения Данных на Диске

Векторные индексы кодовых баз полностью изолированы друг от друга и сохраняются централизованно в родительской директории относительно открытого рабочего пространства. Это предотвращает избыточную нагрузку на системный диск `C:\` и исключает случайную порчу индексов при очистке локального каталога проекта.

### 4. Алгоритм Изоляции Контекста (Кодовая логика)

Разделение данных происходит динамически в модуле инициализации сессий (`src/core/indexer.py`) с помощью функции генерации уникального пути:

```python
def _generate_unique_db_path(project_path: Path) -> Path:
    # 1. Нормализация пути под стандарты и капризы ОС Windows
    normalized_path = str(project_path.resolve()).lower().replace('\\', '/')
    
    # 2. Вычисление уникального 8-символьного MD5-хеша от абсолютного пути
    project_hash = hashlib.md5(normalized_path.encode()).hexdigest()[:8]
    project_name = os.path.basename(project_path)
    
    # 3. Привязка к родительскому каталогу (вынос индекса из папки проекта)
    project_root = project_path.parent
    db_dir = project_root / ".codebase_indices" / "lancedb_v2"
    
    # 4. Формирование финального имени изолированной папки LanceDB
    db_name = f"index_{project_name}_{project_hash}.db"
    return db_dir / db_name

```
